## Bugs found + fixed (executed-verified)
1. **Redirects silently followed (SPEC violation).** SPEC says
   `follow_redirects=False`, but urllib follows 3xx by default. A 302 → valid
   XML would have been ingested from the wrong URL. Fixed with a `_NoRedirect`
   HTTPRedirectHandler in `network.py`; a 3xx now surfaces as an HTTPError in
   `errors[]`, never parsed as feed XML. (`network.py`)
2. **Non-2xx mis-classified as ParseError.** 404/5xx were thrown at the XML
   parser. `fetch_one` now short-circuits non-2xx as an HTTP error and carries
   `error_code` through to the run-result `errors[]` (matches SPEC error
   schema's `error_type`/`error_code`). (`network.py`, `core.py`)
